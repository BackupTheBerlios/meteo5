Plan de développement et test

1. Ordre d'implémentation des classes

Nous avons choisis de commencer à implémenter tout d'abord les classes qui sont les moins dépendantes puis ensuite les classes qui le sont plus. Nous implémenterons d'abord pour chacune de ses classes les interfaces et les classes dont elle hérite.


L'ordre d'implémentation des classes sera le suivant :

  1. IWordAlgo
  2. ToStringAlgo
  3. GuessCharAlgo
  4. IsAllVisibleAlgo
  5. AWordState
  6. InvisibleState
  7. VisibleState
  8. IWordFactory
  9. WordFactory
  10. IWord
  11. IEmptyWord
  12. EmptyWord
  13. INEWord
  14. NEWord

2. Identification des classes faisant l'objet d'un test unitaire

On effectuera aucun test sur les interfaces et sur les classes abstraites car elles ne sont pas instanciables. Nous avons choisis de ne pas effectuer de test sur la classe EmptyWord car elle est très simple.


Les classes sur lesquelles il est nécessaire d'effectuer des tests unitaires sont les suivantes :

  1. ToStringAlgo
  2. GuessCharAlgo
  3. IsAllVisible
  4. InvisibleState
  5. VisibleState
  6. WordFactory
  7. NEWord

L'ordre dans lequel les classes instrumentées ont été énumérées précédement sera l'ordre suivit pour les tester. Il s'agit ici aussi d'une question de dépendance.


3. Identification des principaux scénarios de test prévu

Liste des scénarios de tests unitaires prévus pour chacune des classes suivantes :

  1. ToStringAlgo
    • Vérifier que l'algorithme renvoie bien des lettres cachées lorsque qu'on lui passe des NEWord non visible
    • Vérifier qu'un NEWord visible est correctement affiché (bonne lettre, bon emplacement)
    • Vérifier qu'un EmptyWord renvoie bien une chaîne vide
  2. GuessCharAlgo
    • Vérifier que l'algorithme renvoie vrai lorsqu'il est appliqué à un NEWord visible
    • Vérifier que l'algorithme renvoie vrai lorsqu'il est appliqué à un NEWord invisible et le rend inivisible
    • Vérifier que l'algorithme renvoie faux dans le cas d'un EmptyWord
    • Vérifier que l'algorithme renvoie faux dans le cas d'un NEmptyWord non présent dans le mot
  3. IsAllVisible
    • Vérifier que l'algorithme renvoie vrai sur un mot vide
    • Vérifier que l'algorithme renvoie faux sur un mot invisible
    • Vérifier que l'algorithme renvoie faux sur un mot partiellement visible
    • Vérifier que l'algorithme renvoie vrai sur un mot totalement visible
  4. InvisibleState
    • Vérifier que la méthode toggleState change l'état d'un NEWord invisible à visible
  5. VisibleState
    • Vérifier que la méthode toggleState ne change pas l'état d'un NEWord visible
  6. WordFactory
    • tester que makeWord() crée bien le bon mot
  7. NEWord
    • ToStringAlgo
    • ToStringAlgo
